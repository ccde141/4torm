# 介绍

4torm 是一个多 Agent 协作框架，可以创建多个长期保留的 Agent，为它们配置模型、提示词、工具与技能，再根据任务需要安排到不同功能区中

这些 Agent 可以独立对话、共同讨论、在工作室中长期协作、按工作流接力，或由自动化任务定时唤醒。已经配置的能力可以持续使用，不需要在每次任务中重新创建角色

## 五种协作模式

| 模式 | 代号 | 用途 |
|------|------|------|
| 对话 | 季风 Chat | 单 Agent 多轮对话、工具调用与子任务委托 |
| 会议 | 对流 Convection | 多个 Agent 围绕议题依次讨论，由人类主持并提供会长私聊 |
| 工作室 | 气旋 Cyclone | 设置长期工位，支持工位私聊、多个群聊和共享工作区 |
| 工作流 | 信风 TradeWind | 通过可视化画布安排 Agent 接力、分支与汇合 |
| 自动化 | 潮汐 Tide | 按时间间隔触发 Agent 执行任务并滚动归档 |

对流与气旋都支持多 Agent 讨论，但用途不同：**对流用于围绕具体议题组织会议**，**气旋用于让固定工位长期协作**

## 模式选择参考

```text
要 Agent 帮你完成一件事？                   → 季风（对话）
要几个 Agent 围绕议题共同讨论？             → 对流（会议室）
要一批固定角色长期协作、单聊或开群？         → 气旋（工作室）
要多个 Agent 按流程接力、分支或汇合？         → 信风（工作流）
要 Agent 定时自动巡检或汇报？                → 潮汐（自动化）
```
