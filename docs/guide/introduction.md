# 介绍与设计哲学

4torm 是一个**本地部署**的多 Agent 协作平台。你可以创建多个 Agent,让它们独立对话、组队群聊、按工作流接力,或按时间表自动执行任务。

## 设计哲学

4torm 的出发点很简单:感觉 Agent 更可以像一种员工,而不是一个又一个用完就扔的对话窗口。

正如一个公司的组织方式——老板招了一帮人,每个人有自己的专长、自己的工位、自己的工作习惯。有的活儿一个人干就行,有的得拉几个人开会讨论,有的得按流程一步步接力,有的得每天定时巡查。人是固定的那批人,变的只是"今天用什么方式协作"。

因此我也希望让 AI Agent 也这样工作。

4torm 里的 Agent 不是某次任务里临时捏出来的角色——它是你认真配好提示词、选好模型、装好工具之后长期存在的一个"人"。今天它在对话里帮你查资料,明天同一个它被拉进工作流做审校,后天它被定时任务唤醒写日报。能力配一次,哪里需要往哪里调。

这就是 **复用**。

另一个是 **灵活**。很多问题不一定都只用对话框就能完美完成,真实的工作不一定只有一种形态,不可能所有的事情都靠一个对话窗口搞定,也不可能所有事情都能画成流程图完美实现。因此 4torm 想做的就是把这些能力收拢到同一套 Agent 体系下:同一批人,只不过按需切换不同的协作方式而已。

## 平台定位

- **多 Agent** —— 不止一个 Agent,它们可以分工、讨论、委托
- **多模式** —— 从简单对话到复杂工作流,按需选择协作深度
- **可扩展** —— 工具、技能与 MCP 系统让 Agent 持续获得新能力

## 五种协作模式

| 模式 | 代号 | 一句话 |
|------|------|--------|
| 对话 | 季风 Chat | 单 Agent 多轮对话 + 工具调用 + 子任务委托 |
| 会议 | 对流 Convection | 一次性多 Agent 圆桌会议,人类主持,会长私聊参谋 |
| 工作室 | 气旋 Cyclone | 常驻工作室:角色「工位」可单独私聊,按需开多个群聊房间,共享工作区 |
| 工作流 | 信风 TradeWind | 可视化画布编排 DAG,节点串/并联执行 |
| 自动化 | 潮汐 Tide | 定时触发,Agent 自主执行,滚动归档 |

> 对流与气旋都是多 Agent 群聊,但定位不同:**对流是一次性会议**(开完即走),**气旋是常设工作室**(固定角色工位、可单独私聊、可同时开多个群聊房间、共享一块工作区)。

### 模式选择参考

```
要 Agent 帮你做一件事?               → 季风(对话)
要几个 Agent 临时开会讨论出结论?     → 对流(会议室)
要一批固定角色长期协作、能单聊也能开群? → 气旋(工作室)
要多个 Agent 按顺序接力完成?         → 信风(工作流)
要 Agent 定时自动巡检 / 汇报?        → 潮汐(自动化)
```
