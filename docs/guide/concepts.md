# 核心概念

无论用哪种协作模式,4torm 底层都共享同一套概念。理解这几个,五种模式就都通了。

## Agent · 持久实体

Agent 是平台的基本工作单元,也是被「复用」的那个"人"。每个 Agent 拥有:

- **角色提示词**(`role-prompt.md`)—— 定义身份、能力边界、行为规范(引擎运行时读取的真理来源)
- **模型配置** —— 指定 LLM 提供商和模型、温度等
- **工具列表** —— 可调用的工具集合
- **技能列表** —— 可按需加载的专业知识模块
- **独立工作区** —— 文件读写的沙箱目录

创建后长期存在,可在任意模式间复用:今天在季风对话,明天进信风工作流,后天被潮汐唤醒。

## 共享 ReAct 引擎

所有协作模式共享同一套 ReAct 循环:

```
接收任务 → 思考(think) → 行动(action / 工具调用) → 观察(result) → 循环
         → 直到输出最终回答(answer)
```

工具调用、子任务委托、流式输出全部统一在这套引擎上。季风是基础,其余模式都复用它:对流是季风的多人版,信风是它的编排层,潮汐是它的时间驱动层,气旋则把私聊与群聊合在一个常驻工作室里。

## 工具 Tool

Agent 与外部世界交互的能力。通过 `data/tools/registry.json` + `data/tools/executors/*.js` 注册。内置文件读写、目录浏览、命令执行、网页抓取等,支持自定义扩展。详见[工具制作](../extend/tools)。

## 技能 Skill

Agent 的领域专长模块。通过 `data/skills/{id}/SKILL.md` 定义专业提示词,Agent 用 `use_skill` 工具按需加载,不占用常驻上下文。可携带专属工具。详见[技能制作](../extend/skills)。

## MCP · 外部能力接入

Model Context Protocol。在「MCP」页接入外部 MCP 服务器(stdio + JSON-RPC),其工具以 `mcp:服务名:工具名` 注入统一工具池,Agent 用 `mcp:服务名:*` 通配即可调用。详见 [MCP 接入](../extend/mcp)。

## 长期记忆触发

Agent 工作区下的 `.workspace/MEMORY.md` 文件,在用户消息命中「记忆 / 记住 / 之前 / 上次 / 历史 / 回忆 / 还记得」等关键词时自动注入对话上下文。让 Agent 跨会话"记得"事情。

## 流式输出全程贯通

LLM 流式 token、工具调用、子任务派发——全程 SSE(`text/event-stream`)推送到前端。会话窗口关闭再打开能接上正在进行的流(信风节点和会议室节点都支持)。

## 沙箱级别隔离

Agent 文件操作受沙箱限制,分三档:

| 级别 | 约束 |
|------|------|
| `strict` | 最严,仅限自身工作区 |
| `relaxed` | 默认,放宽到约定的可访问范围 |
| `unrestricted` | 不限制路径 |

按级别约束 Agent 可访问的文件路径。详见[安全与隔离](../architecture/security)。
