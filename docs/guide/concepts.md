# 核心概念

无论选择哪种协作方式，4torm 都建立在同一套基础概念之上。理解 Agent、工具、技能、MCP、记忆和执行权限后，也就能理解五个功能区如何共同工作。

## Agent · 持久实体

Agent 是 4torm 的基本工作单元，也是能够在不同功能区中反复使用的长期角色。每个 Agent 拥有：

- **角色提示词**（`role-prompt.md`）—— 定义它的身份、职责和工作方式
- **模型配置** —— 指定使用的模型、提供商和温度等参数
- **工具列表** —— 决定它能够调用哪些工具
- **技能列表** —— 提供可以按需加载的专业知识和工作方法
- **独立工作区** —— 保存该 Agent 长期使用的文件

Agent 创建后会长期保留，可以今天参与季风会话，明天进入信风工作流，之后再由潮汐任务定时唤醒。它的基础能力不需要在每个功能区中重新配置。

## 工具 Tool

工具负责 Agent 与外部环境的实际交互，包括文件读写、目录浏览、命令执行和网页获取等能力。框架内置了一组常用工具，也可以通过 `data/tools/registry.json` 和 `data/tools/executors/*.js` 添加自定义工具，具体制作方式见[工具 · 使用与制作](../extend/tools)。

## 技能 Skill

技能用于保存可以反复使用的专业知识与工作方法，其主要内容位于 `data/skills/{id}/SKILL.md`。Agent 会在需要时通过 `use_skill` 加载相应说明，不会让所有技能始终占用会话上下文；技能也可以附带完成相关任务所需的专用工具，详见[技能 · 使用与制作](../extend/skills)。

## MCP · 外部能力接入

MCP（Model Context Protocol）用于接入外部工具服务。通过 MCP 页面，可以使用 stdio、Streamable HTTP 或兼容 SSE 连接搜索、浏览器、数据库等服务；连接成功后，还需要在 Agent 配置中明确选择要使用的工具，详见[MCP · 外部工具接入](../extend/mcp)。

## 长期记忆触发

Agent 工作区中的 `.workspace/MEMORY.md` 用于保存长期记忆。当用户消息中出现「记忆」「记住」「之前」「上次」「历史」「回忆」或「还记得」等表达时，系统会将这份记忆加入当前上下文，帮助 Agent 在不同会话之间延续信息。

## 流式输出

模型生成的内容、思考过程、工具调用和子任务委托都会通过 SSE（`text/event-stream`）持续发送到前端。关闭会话窗口不会中断正在运行的任务，重新打开后可以继续查看当前进度；信风节点和会议室节点也使用相同的实时状态机制。

## 执行权限

Agent 提供两种执行权限：

- **项目级**：内置文件工具可以访问 4torm 项目和当前工作区。工作区即使位于 4torm 目录之外，也可以正常使用；其他未配置的外部路径会被拒绝
- **无限制**：内置文件工具可以访问任意路径，适合需要跨目录处理文件的 Agent

无论选择哪种权限，普通文件工具都不能直接改写 Agent 注册表、潮汐任务表和工作流控制文件，这些内容需要通过对应功能入口或专用工具修改。

执行权限主要约束框架内置文件工具，命令、MCP 工具和自定义执行器仍按照各自具备的能力运行，具体边界见[安全与隔离](../architecture/security)。
