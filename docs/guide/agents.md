# 控制台与 Agent 管理

## 控制台是什么

控制台是所有 Agent 的管理入口。于此创建的 Agent 会长期保留，并可以在之后的季风·会话、对流·会议室、气旋·工作室、信风·工作流和潮汐·自动化这几类功能区中复用。你配置的是同一个 Agent，而不是某个会话里的临时角色。

每张 Agent 卡片会显示它的角色、默认模型、工作区和当前状态。页面顶部汇总 Agent 总数、空闲实例数和会话数；状态与自定义标签可以帮助筛选，但不会改变 Agent 的任何能力。

如果 Agent 绑定的模型已经不可用，例如对应的模型提供商被删除，卡片会显示异常状态。Agent 本身及其数据仍会保留，重新绑定有效模型后即可继续使用。

## 创建 Agent

点击控制台右上角的「创建 Agent」即可进入配置页面。创建时不必一次配置完所有能力，之后可以随时通过 Agent 卡片右上角的设置按钮重新调整。

名称、角色和描述主要用于识别：

- **名称**是 Agent 在各功能区中显示的名字。
- **角色**是一句简短的身份标记，例如“程序员”或“资料整理员”，仅用于在卡片上标注，对 Agent 的工作能力和执行方式没有影响。
- **描述**用于补充它负责什么，方便在 Agent 较多时区分。
- **分类标签**只用于控制台筛选，不会进入 Agent 的提示词，也不会改变运行方式。

默认模型决定 Agent 通常使用哪个模型。建议在创建时明确选择；留空时，系统会使用当前已注册模型列表中的第一个模型。

「角色提示词」主要决定 Agent 如何理解自身身份和任务方式；卡片上的“角色”只是用于界面分类与展示的备注。

工作区留空时，系统会为 Agent 创建独立目录。

执行权限会随 Agent 保存，并在各功能区调用内置文件工具时自动生效；对流、气旋和信风使用的共享工作区会自动纳入允许范围。具体边界见[安全与隔离](../architecture/security)。

## 配置工具与技能

工具决定 Agent 能执行哪些操作，技能则提供完成某类任务时需要遵循的方法和知识。两者可以独立配置，技能也可以附带自己的专用工具。

![Agent 配置页面](/images/guide/agent-configuration.webp)

创建 Agent 时，框架内置工具会默认启用，使 Agent 可以直接读写文件、执行命令和查找网页。如果只想让它进行对话，可以手动取消全部本地工具；`ask`、`task_board` 和 `delegate` 等维持完整会话所需的能力仍会保留。

工具列表遵循以下规则：

- 保存后，页面上的选择就是 Agent 的本地工具清单，未选中的工具不会提供给 Agent。
- **自定义工具和 MCP 工具不会默认启用，需要明确选择。**
- 技能附带的专用工具会随技能自动加入，不需要在全局工具列表中重复选择。

新建 Agent 时，当前已经安装的技能会默认启用，可以在保存前手动取消。技能的 `SKILL.md` 不会一直放在 Agent 的上下文中；Agent 会在需要时调用 `use_skill`，临时读取对应技能的说明。只要技能仍处于启用状态，系统就会保留这一加载能力。

工具名称旁的警示标记只表示它具备写文件、删除内容或执行命令等能力，不代表每次调用前都会再次弹出确认。
