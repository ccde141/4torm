# 潮汐 · 自动化

潮汐用于让 Agent 按固定间隔重复执行任务。每个潮汐任务会绑定一名已注册的 Agent，并设置任务内容、触发间隔和执行次数；到达触发时间后，系统会自动唤醒 Agent，并保存本次回复、工具调用和运行结果。

Agent 每次运行时都会读取控制台中的当前模型、工具、技能和执行权限配置。内置文件工具使用该 Agent 的工作区。任务可以维护一份独立的潮汐会话，也可以将回复追加到指定的季风会话：前者适合长期巡检、资料整理等独立自动化，后者适合把定期结果持续汇入已有对话。

调度由服务端负责，不需要一直停留在潮汐页面。4torm 关闭期间任务不会执行；重新启动后，调度器会根据已经保存的任务配置和下次触发时间继续工作。

潮汐运行期间不提供全局工具注册。任务只能使用触发时已经由 Agent 配置启用的工具和技能；需要新增工具时，应先在季风会话或气旋工位私聊中完成创建与确认。

## 创建任务

在左侧选择一名 Agent，再点击任务列表旁的加号即可创建潮汐任务。任务会绑定当前选中的 Agent；创建后可以修改任务名称、执行间隔、任务内容和剩余次数，但不能直接更换 Agent。对于需要长期运行的自动化，建议创建专用 Agent，便于区分其会话、状态和工作区内容。

创建时需要设置以下内容：

- **任务名称**：用于任务列表和运行记录中的识别。
- **执行间隔**：分别设置时、分、秒，三项不能同时为零。
- **任务内容**：每次触发时发送给 Agent 的指令。
- **重复次数**：填写正数表示计划执行次数，`-1` 表示持续运行；任务创建后显示的是剩余次数。
- **推送目标**：决定任务使用独立潮汐会话，还是将回复写入已有季风会话。
- **保留轮数**：决定独立潮汐会话保留多少轮上下文。设为 `1` 时，每轮都不读取此前内容，并在完成后单独归档；设为大于等于 `2` 的偶数时，会话达到该轮数后自动归档较早的一半，并保留最近一半供后续运行使用。
- **自循环模式**：允许 Agent 在每轮结束时决定下一轮任务内容。启用后，任务会固定使用独立潮汐会话、保留两轮上下文并持续运行。

任务创建后会立即启用，并从当前时间开始计算第一次触发时间；创建本身不会立刻执行任务。任务卡片会显示当前状态、剩余次数、上次运行时间和下次触发时间。

## 推送目标与会话

推送目标决定任务运行时从哪里读取上下文，以及回复最终保存到哪里。创建任务时可以选择「累积潮汐会话」或「指定季风会话」。

### 独立潮汐会话

「累积潮汐会话」是默认方式。每个任务拥有自己的潮汐会话，任务内容、Agent 回复和工具调用都会保存在其中，不会混入该 Agent 的季风会话。

后续运行可以根据保留轮数读取此前内容，使 Agent 延续上一次任务的进度。达到设定的轮数后，较早内容会自动归档，避免活跃上下文持续增长。

这种方式适合巡检、资料整理、定期生成内容，以及其他不需要进入日常对话的长期任务。

### 指定季风会话

「指定季风会话」会将潮汐任务接入该 Agent 已有的一段季风对话。每次触发时，Agent 会读取这段会话的已有上下文，再将本轮任务内容和最终回复追加到其中。

目标季风会话只会追加本轮任务内容和最终回复，工具调用过程不会出现在该会话中；服务端仍会将其保存在对应的潮汐运行记录文件中。该方式不使用潮汐的保留轮数和自动归档机制，因此会话的上下文会持续增长。

如果目标季风会话当时正在回复，本次潮汐运行会记录为失败，不会修改会话，也不会立即重试。

多个潮汐任务可以指向同一季风会话，但不能同时写入；发生冲突时，后到任务同样会记录失败，并等待下一次计划时间。

## Rolling Window 与自动归档

Rolling Window（滚动窗口）只作用于独立潮汐会话。「保留轮数 N」决定当前会话保留多少轮近期内容，避免长期任务不断累积上下文。归档内容仍保存在本地，但不会继续发送给模型。

- **N = 1**：每轮独立运行，不读取上一轮内容；本轮结束后，完整记录立即归档。
- **N ≥ 2**：N 必须是偶数。会话达到 N 轮时，较早的 N/2 轮会被归档，最近的 N/2 轮继续作为后续上下文。

例如将 N 设置为 `8`：完成第 8 轮后，第 1 至 4 轮进入归档，第 5 至 8 轮留在当前会话；再完成四轮后，窗口会再次向前滚动。

这种方式不会在归档时突然清空全部上下文，同时能够限制长期任务发送给模型的历史长度。

## 自循环模式

普通潮汐任务每轮执行相同的任务内容；启用自循环后，Agent 会在完成本轮工作时给出下一轮需要处理的具体任务。最初填写的任务内容会始终作为最高目标保留，后续任务只能在这个目标下继续演进。

系统会自动提取 Agent 给出的下一轮任务，并更新任务内容；这段交接指令不会显示在最终回复中。下一次触发时，Agent 会同时收到最初目标和最新一轮任务。

启用自循环后，任务会固定使用独立潮汐会话、两轮滚动窗口和永续运行。任务不会因为 Agent 判断“工作完成”而自行结束，需要停止时应手动暂停。

如果 Agent 没有给出有效的下一轮任务，系统会保留当前任务内容，并在下一次触发时继续执行。

## 管理任务

任务卡片会显示当前状态、剩余次数以及上次和下次运行时间。点击卡片可以查看完整配置和最近五次运行记录。

### 暂停与编辑

暂停任务只会阻止后续自动触发，不会中止已经开始的本轮执行。任务暂停后，可以修改名称、执行间隔、任务内容、保留轮数和剩余次数。

重新启用任务时，下次触发时间会从启用时刻重新计算，不会立即补跑暂停期间错过的轮次。

### 立即运行

「立即运行」用于提前触发一次任务。同一任务已经执行时不会重复启动。

任务处于启用状态时，立即运行视为一次正常触发，会扣减剩余次数并重新计算下次触发时间；任务处于暂停状态时，则只进行一次手动运行，不扣减次数，也不改变原定计划。

### 会话与运行记录

任务详情会显示最近五次运行的成功或失败状态、耗时、工具调用轮数和回复摘要。点击「查看会话」可以打开该次运行对应的完整会话。

选择 Agent 后，左侧会话列表会显示它的独立潮汐会话。指定季风会话不会重复出现在这里，但仍可以从运行记录进入查看。

### 删除任务与会话

删除任务会清除任务配置、运行记录，以及该任务自己的潮汐会话和归档。任务正在执行时不能删除，需要等待本轮结束。

删除指向季风会话的任务不会删除对应的季风会话。

独立潮汐任务暂停后，还可以只删除当前活跃会话并保留归档。任务再次运行时会创建新的活跃会话。

## HTTP API

所有端点前缀 `/api/tide`。

| 方法 | 路径 | 功能 |
|------|------|------|
| `GET` | `/tasks` | 列出所有任务 |
| `POST` | `/task` | 创建任务 |
| `GET` | `/task/:taskId` | 任务详情 + 近 5 条运行记录 |
| `PATCH` | `/task/:taskId` | 修改任务 |
| `DELETE` | `/task/:taskId` | 删除任务及归档 |
| `POST` | `/task/:taskId/toggle` | 启用 / 暂停切换 |
| `POST` | `/task/:taskId/run-now` | 立即执行一次 |
| `GET` | `/task/:taskId/runs?limit=20` | 运行历史 |
| `GET` | `/sessions/:agentId` | Agent 的潮汐会话摘要 |
| `GET` | `/session/:agentId/:sessionId` | 读取会话完整消息 |
| `DELETE` | `/session/:taskId` | 删除活跃会话(保留 bak 归档) |

## 失败与暂停

每次运行的成功或失败都会保存在运行历史中。失败记录会显示具体原因，方便判断是 Agent、模型、工具还是目标会话的问题。

单次失败不会立即停用任务，任务仍会等待下一次计划时间。连续失败三次后，系统会自动暂停任务，并在任务详情中显示暂停状态。修复原因后，可以手动重新启用。

如果指定的季风会话正在被其他回复占用，本次潮汐运行会记录失败，但不会修改该会话，也不会计入连续失败次数。

## 数据目录

```
data/tide/
├── tasks.json              ← 所有任务配置(TideTask[] 数组)
├── tasks.template.json     ← 空模板
└── runs/
    └── {taskId}/
        ├── 2026-06-04T03-09-27.573Z.json   ← 每次运行记录
        └── ...
```

### 任务配置类型

```typescript
interface TideTask {
  id: string                // 自动生成,格式 "tide-{时间戳36}-{随机6}"
  name: string
  schedule: string          // "every 5m" / "every 2h30m" / "every 30s"
  prompt: string
  agentId: string
  repeatCount: number       // -1=永续, N=剩余次数, 0=结束
  pushMode: 'accumulate' | 'designated'
  targetSessionId?: string
  windowN: number           // 1 或 ≥2 的偶数
  selfLoop: boolean
  consecutiveErrors: number
  enabled: boolean
  createdAt: string         // ISO 格式
  lastRun?: string
  nextRun?: string
}
```
