# 对流 · 会议室

对流用于人类与多个 Agent 共同讨论。人类发言后，参与者会按照设定顺序依次回复；后发言的 Agent 可以看到此前的会议内容和本轮已有回复，使不同角色能够在同一上下文中继续讨论。

每个会议室还设有一名会长。会长不参与公共发言，而是在右侧私聊侧板中提供单独回应。会长能够参考公共会议记录，但私聊内容不会进入公共讨论。

参与者可以使用各自配置的工具，并共同操作当前会议室的工作区。会议记录、成员配置和工作区都由会议室单独保存。

对流会议不提供全局工具注册。参与者可以使用各自已经配置的工具，但不能在公共讨论、会长私聊或会议轮次中新增工具；相关需求可以转到季风会话或气旋工位私聊处理。

## 创建会议室

创建会议室至少需要两个 Agent。点击左侧会话列表右上角的加号后，系统会默认选择第一个 Agent 作为会长，并从其余 Agent 中选择最多两名参与者。

创建后，可以在会议顶部更换会长，增加或移除参与者、调整参与者的发言顺序；这些设置会决定下一轮人类发言后参与回复的 Agent 及其顺序。会议标题可以双击修改；删除会议室时，其公共记录、会长私聊和会议工作区会一并删除。

## 进行公共讨论

在底部输入框发言后，会议会开始一轮回复。参与者按照顶部排列的顺序依次运行，每名 Agent 都能看到此前的公共记录，以及本轮中排在它之前的 Agent 给出的最终回复。

参与者的思考过程和工具调用会实时显示给用户，但不会进入其他 Agent 看到的公共记录；只有最终回复会成为后续讨论的上下文。会议进行期间仍可继续发送内容，新消息会排队到下一轮；点击停止按钮可以结束当前轮次。

## 会长私聊

右侧侧板用于与会长单独交流。会长会根据发起私聊时的公共会议记录和已有私聊内容作出回应；私聊内容不会进入公共讨论，也不会被其他参与者看到。

公共讨论与会长私聊按顺序运行。一侧正在回复时，另一侧仍可编辑草稿，但需要等待当前轮结束后再发送。会长私聊只生成文字回复，不会调用工具。

## 动态管理

会议进行过程中，可以随时增加、移除参与者，或调整他们的发言顺序。修改不会干扰正在运行的轮次，而会从下一轮公共讨论开始生效。

如果下一轮只需要某一名 Agent 回应，可以暂时只保留该参与者；需要恢复多人讨论时，再将其他 Agent 加回会议。会长也可以随时更换，并从下一次会长私聊开始生效。

## 工具与工作区

参与者会使用各自在控制台中配置的内置工具、自定义工具、技能工具和 MCP 工具。所有参与者共同操作当前会议室的工作区；先发言者产生的文件或修改，后续参与者可以继续读取和处理。

参与者调用内置文件工具时，会自动应用各自 Agent 的执行权限；会议室工作区始终作为当前工作区纳入允许范围。

思考过程和工具执行会显示在对应的回复气泡中，但不会进入其他参与者看到的公共记录。对流中的参与者不能创建 SubAgent；会长私聊也不会调用工具。

## 重置会话

需要清理当前上下文时，可以使用 `/reset` 和 `/reset summary` 两个指令：

- `/reset` 会先归档公共讨论和会长私聊，再清空两处消息；会议成员配置和工作区文件不受影响。
- `/reset summary` 同样会先归档并清空两处消息，再使用会长的模型根据公共讨论生成一份结构化摘要，将其作为新上下文的起点。如果摘要生成失败，界面会明确提示，原始记录仍会保留在归档中。

会话归档文件会按顺序保存在当前会议工作区的 `bak/` 目录中。

## 自动压缩

当公共讨论的上下文达到 `200K prompt tokens`，并且已有足够的讨论轮次时，系统会在当前轮所有参与者回复完成后自动整理记录。

压缩时，较早的讨论会先归档到会议工作区的 `bak/` 目录，再由会长的模型生成结构化摘要；最近的若干轮会保留原文。摘要与这些近期记录将共同成为后续讨论的上下文，界面会显示当前压缩状态和结果。

如果摘要生成失败，原有公共记录不会被替换，界面会显示错误；当前会议之后不会继续自动尝试压缩，避免每轮重复失败。

## 数据与恢复

会议记录、成员配置和工作区均保存在本地。刷新页面后，系统会重新打开上次使用的会议室；如果当时仍有轮次正在运行，页面会同步其运行状态，并在轮次结束后重新载入最终记录。刷新期间已经产生的实时输出和工具过程不会重新播放。

对流数据位于以下目录：

- 会话索引：`data/convection/sessions/_index.json`
- 会话记录：`data/convection/sessions/{sessionId}.json`
- 会议工作区：`data/convection/sessions/{sessionId}/workspace/`
- 归档记录：`data/convection/sessions/{sessionId}/workspace/bak/`

其中，`data/convection/sessions/` 目录已被 Git 忽略，不会随代码提交。Git 忽略不等于数据备份；需要迁移或保留会议时，请记得单独备份该目录。
