# 气旋 · 工作室

气旋用于让多个 Agent 在同一个长期工作空间中协作。工作室由若干工位组成；每个工位指向一个已注册的 Agent，并保存独立的职责、角色提示词和长期私聊记录。工位绑定的 Agent 可以随时更换。

你可以直接与某个工位交流，也可以随时创建群聊，让选中的工位按照顺序共同讨论。所有工位和群聊共享工作室文件，但各自的对话上下文彼此独立。

每个工位私聊还带有一块独立任务板，用于记录当前工位的任务目标和进度。其使用方式与季风的[会话任务板](./chat#会话任务板)一致。

工作室内设有一份由所有 Agent 共享的公告板，用于长期信息、公共约定和任务目标的共享。

工位之间，以及群聊中的 Agent 与工位之间，可以通过两种方式联络：

- `contact` 用于同步联络。当前 Agent 向目标工位询问信息，并等待其回复后继续当前任务。
- `dispatch` 用于异步委托。当前 Agent 将需要较长时间处理的任务交给目标工位，无需等待结果即可继续当前讨论或工作。

工作室还可以设置一名会长。会长不占用工位，也不参与群聊发言，而是在每个群聊的右侧私聊侧板中提供单独回应。其使用方式与[对流会议室中的会长](./convection#会长私聊)一致。

## 创建工作室

点击左侧工作室列表右上角的加号，填写工作室名称即可创建。会长可以在此时选择，也可以留空，之后再从任意群聊的会长侧板中设置。

新建的工作室暂时没有工位和群聊。你可以先添加需要长期保留的工位，再根据具体议题创建群聊。工作室的名称、会长、工位、群聊和共享文件都会独立保存。

## 创建和配置工位

点击工位列表旁的加号即可创建工位。每个工位需要选择一个已注册的 Agent，并设置工作室内唯一的工位名称。角色提示词和职责名片可以留空，之后再通过工位卡片上的设置按钮补充。

角色提示词决定 Agent 在当前工位中的具体身份和工作方式。默认情况下，它会叠加在 Agent 原有的角色提示词之后；开启「覆盖 Agent 基础角色」后，则只使用当前工位的角色提示词。

职责名片是一句提供给其他工位的职责说明，主要用于 `contact` 和 `dispatch` 时识别目标，也可以根据 Agent 与工位配置自动生成。

工位绑定的 Agent、名称、角色提示词和职责名片都可以随时修改。更换 Agent 后，新绑定会从下一轮开始生效，工位已有的配置和私聊记录不会被清空。如果原本绑定的 Agent 已被删除，重新绑定一个有效 Agent 即可继续使用。

### 工位私聊

每个工位存在单一会话。点击工位卡片即可进入；Agent 会使用当前绑定的模型、工具和技能，并按照该工位的角色提示词工作。文件读写发生在工作室的共享目录中。

工位调用内置文件工具时，会自动应用当前绑定 Agent 的执行权限；工作室共享目录始终作为当前工作区纳入允许范围。

思考过程、工具调用、SubAgent 委托和当前执行状态都会显示在会话信息中。

工位私聊的上下文不会进入群聊，也不会与其他工位混合。Agent 可以通过 `contact` 向其他工位同步询问，也可以通过 `dispatch` 将复杂任务异步交给其他工位。

工位私聊也允许 Agent 创建并注册新的全局工具。注册前会在当前私聊中请求确认，确认后才会写入全局工具列表。

这项能力只属于工位私聊。群聊、会长私聊，以及由 `contact` 或 `dispatch` 触发的无人联络回合不会注册工具；公共讨论产生相关需求时，可以将任务交给固定工位，再由人类在该工位私聊中确认。

## 创建和管理群聊

点击群聊列表旁的加号即可创建群聊。创建时可以设置名称、讨论话题、运行模式和初始参与工位；暂时不选择工位也可以创建，之后再添加。

每个初始工位可以选择一种入会方式：

- **工作摘要**：根据该工位近期的私聊进展生成一段公开摘要。
- **自我介绍**：根据工位职责生成一段简短介绍。
- **静默入会**：加入群聊，但不产生入会发言。

入会发言会成为群聊记录的一部分，但不会将工位的完整私聊上下文带入群聊。

群聊提供两种运行模式：

- `build`：参与者可以使用控制台配置的工具，并读写工作室共享目录。
- `plan`：参与者不能直接调用写文件、删除或命令执行等工具，但仍可以读取资料，并使用 `contact` 和 `dispatch` 联络其他工位。

创建后，可以在群聊顶部增加、移除或调整工位顺序。参与者会在下一轮人类发言后按照当前顺序依次回复。群聊正在运行时暂时不能修改成员与顺序；当前轮结束后即可调整，并从下一轮开始生效。

### 工位互联

群聊中的参与工位可以联络工作室里的固定工位，把公共讨论转成具体执行。

`contact` 用于同步联络。当前 Agent 向目标工位发送问题或补充信息，并等待目标工位回复后继续当前回复。联络内容会进入目标工位的私聊记录；目标工位的回复只作为本次工具调用结果返回，不会自动加入群聊公共记录。

`contact` 比较适合询问进展、核对信息或请求一个可以快速完成的判断。目标工位正忙、正在等待人类回答，或联络链形成循环时，系统会直接返回失败原因，当前回复不会无限等待。

`dispatch` 用于异步委托。当前 Agent 将需要较长时间处理的任务交给目标工位后即可继续群聊；目标工位会在自己的私聊中独立执行，执行状态和结果会形成派发记录。派发完成后，人类可以决定是否将结果带回当前群聊。

两种联络都共享工作室文件，但不会合并不同工位的私聊上下文。

## 会长私聊

每个工作室可以设置一名会长。会长可以从任意群聊右侧的私聊侧板中设置、更换或清空；这项设置属于整个工作室，会同时作用于其中的所有群聊。

每个群聊保存独立的会长私聊记录。会长能够参考当前群聊近期的公共讨论、在场工位和工作室公告板，但看不到其他群聊或工位私聊。会长私聊也不会进入公共讨论。

会长只提供文字分析和建议，不会调用工具。公共讨论与会长私聊不能在同一群聊中同时运行，一侧结束后才能启动另一侧。

更换会长会从下一次私聊开始生效，已有的会长私聊记录不会被清空。

## 重置会话

重置指令需要作为一条消息，输入到需要处理的会话输入框中。指令作用范围由输入位置决定。

### 群聊公共输入框

在群聊底部的公共输入框中：

- `/reset`：归档并清空当前群聊的公共讨论，会长私聊不受影响。
- `/reset summary`：使用会长模型总结公共讨论，并以摘要作为新的公共上下文。
- `/reset all`：同时归档并清空公共讨论与当前群聊的会长私聊。
- `/reset all summary`：分别总结公共讨论和会长私聊，再重置两处上下文。

### 会长私聊输入框

在右侧会长私聊侧板的输入框中：

- `/reset`：只归档并清空当前群聊的会长私聊。
- `/reset summary`：只总结并重置当前群聊的会长私聊。

### 工位私聊输入框

在工位私聊的输入框中：

- `/reset`：归档并清空当前工位的私聊上下文。
- `/reset summary`：总结当前工位的私聊记录，并以摘要作为新的上下文。

重置不会删除工作室共享文件、工位配置、群聊成员或发言顺序。公共讨论被重置后，此前的异步派发记录仍会保留，但不会再带入新的讨论。归档文件保存在当前工作室的 `bak/` 目录中。

摘要重置需要可用的模型；群聊摘要还需要工作室已经设置会长。如果摘要生成失败，重置不会继续执行，原有记录仍然保留。

## 数据与恢复

工作室配置、工位私聊、群聊记录、公告板、异步派发和共享文件均保存在本地。刷新页面后，已经写入的数据会重新载入；尚未完成的实时输出和工具过程不会作为完整记录重新播放。

气旋数据位于以下目录：

- 工作室索引：`data/cyclone/_index.json`
- 工作室配置：`data/cyclone/{workshopId}/meta.json`
- 共享工作区：`data/cyclone/{workshopId}/workspace/`
- 公告板及修改记录：`bulletin.json`、`bulletin-history.json`
- 工位私聊与任务板：`data/cyclone/{workshopId}/seats/`
- 群聊与会长私聊：`data/cyclone/{workshopId}/rooms/`
- 异步派发记录：`data/cyclone/{workshopId}/dispatches/`
- 会话归档：`data/cyclone/{workshopId}/bak/`

应用重新启动后，尚未开始的异步派发会继续排队执行；关闭时正在执行的派发会被标记为失败，并明确显示关闭原因，不会自动重试。

整个 `data/cyclone/` 目录已被 Git 忽略，不会随代码提交。Git 忽略不等于数据备份；迁移或保留工作室时，需要单独备份该目录。
