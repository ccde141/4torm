# 季风 · 会话

## 季风是什么

季风是单个 Agent 的持续对话界面。你可以在这里与 Agent 交流，也可以让它读取资料、修改文件、执行命令、调用技能，或把部分任务委托给其他 Agent。

同一个 Agent 可以建立多段独立会话。每段会话分别保存消息、工具执行过程、思考内容、任务板和模型选择；Agent 的身份配置、工具能力与工作区则由这些会话共同使用。

调用内置文件工具时，季风使用该 Agent 的工作区，并自动应用它在控制台中保存的执行权限。

## 开始一段会话

进入「季风」后，先在左侧选择 Agent。该 Agent 已有的会话会显示在下方；点击会话区右上角的加号，可以创建一段新会话。

新会话默认使用 Agent 在控制台中绑定的模型。会话顶部的模型选择器可以切换当前会话使用的模型，这项选择不会修改 Agent 的默认模型，也不会影响它在其他功能区中的配置。

发送第一条消息后，会话标题会根据内容自动生成。双击标题可以重新命名；删除会话时需要确认，删除后，该会话的消息与执行记录无法恢复。

## 查看执行过程

发送消息后，回复气泡和输入区上方会实时显示 Agent 当前处于等待响应、生成内容、准备参数还是执行工具，并标出持续时间。能够识别目标时，也会显示正在使用的工具、处理的文件或执行的命令，方便判断长时间任务是否仍在运行。

模型提供思考内容时，思考过程会与正式回复分开显示，并在回复完成后自动收起。工具调用则按执行顺序形成卡片，可以展开查看参数和结果；文件写入与编辑会直接显示改动差异，执行失败时也会保留错误信息。

## 回复途中继续操作

Agent 回复期间仍可继续发送消息，新消息会按顺序排队，并可以在输入框上方撤回。切换到其他会话或 Agent 不会中断当前任务；任务会在后台继续，返回原会话后可以接着查看。需要结束当前运行时，点击停止按钮即可。

## 子任务委托

任务较复杂时，Agent 可以通过 `delegate` 创建临时 SubAgent，交由它独立调查或处理其中一部分工作。SubAgent 使用隔离的对话上下文，并沿用当前会话的模型以及 Agent 的工具和工作区；完成后将结果返回主 Agent，由主 Agent 继续处理。

委托过程会显示为独立卡片，可以查看 SubAgent 的思考、工具调用和最终结果。

## 创建独立工具

季风会话允许 Agent 创建并注册新的全局工具。Agent 会先完成工具执行器和定义，再通过当前会话请求确认；只有你选择「注册」后，工具才会进入全局工具列表。

注册成功不会自动修改 Agent 已经保存的显式工具清单。需要使用新工具时，可以回到控制台为相应 Agent 启用。

## 压缩长会话

会话上下文达到约 8000 token 后，可以发送 `/compact` 生成结构化摘要。系统会压缩较早的对话，同时保留最近四条消息不变，方便在减少上下文占用后继续当前任务。

压缩不会删除原始记录：旧消息仍保留在会话中，并可通过压缩标记查看；后续发送给模型的内容则从最新摘要开始。压缩失败时不会修改会话。压缩完成后，token 统计会暂时等待更新，并在下一次模型回复后重新计算。

## 会话任务板

季风页面右侧的侧板用于显示当前会话的任务板。处理多步骤或者复杂任务时，Agent 会在其中记录目标和进度，任务分为待办、进行中、完成和受阻四种状态。

任务板属于当前会话，不会与同一 Agent 的其他会话混合。展开后可以手动新增、编辑或删除任务，也可以调整任务状态和备注。
