# MCP · 外部工具接入

MCP 用于将外部工具服务接入 4torm，使 Agent 能够使用搜索、浏览器、数据库或其他应用提供的能力，而不需要为每项能力单独编写本地工具。

当前支持三种连接方式：

- **stdio**：由 4torm 启动本地 MCP Server 进程，并通过标准输入输出通信。
- **Streamable HTTP**：连接现代 MCP Server 提供的远程 HTTP 地址。
- **SSE**：兼容仍使用 SSE 传输的旧式远程 MCP Server。

4torm 当前接入 MCP Server 提供的工具，不处理 MCP Resources 和 Prompts。远程服务可以通过自定义请求头使用固定 Token 或 API Key，但暂不处理 OAuth 登录流程。

MCP Server 连接成功后，其工具不会自动提供给所有 Agent，需要进入 Agent 配置页面，选择该服务中的全部工具或单独选择需要的工具。Agent 调用时与本地工具没有区别，但实际能力、数据访问范围和运行结果由对应的 MCP Server 决定。

## 添加 MCP Server

进入侧栏的 MCP 页面，点击右上角的「添加」，选择连接方式并填写配置。服务名称用于页面识别和 Agent 工具分组，创建后不能修改。

使用 **stdio** 连接本地服务时，需要填写启动命令，并按顺序逐项添加启动参数；每一项都是一个完整参数，其中的空格会被保留，还可以按需设置环境变量和工作目录。

使用 **Streamable HTTP** 或 **SSE** 连接远程服务时，需要填写服务 URL。远程服务要求固定 Token、API Key 或其他请求头时，可以在请求头区域逐项添加。

已有 MCP 配置也可以通过 JSON 导入。系统支持单项配置，以及使用 `mcpServers` 包装的多项配置；批量导入时，只有全部配置均有效且名称不冲突才会保存。

保存后，服务默认启用并立即尝试连接。连接成功后，卡片会显示发现的工具数量；连接失败时会保留配置并显示错误，不会自动重试，可以在修正配置后手动重新连接。

## 管理 MCP Server

![MCP Server 管理页面](/images/extend/mcp-server-management.webp)

每张服务卡片会显示连接方式、启动命令或服务地址，以及当前连接状态；连接成功时会显示已经发现的工具数量，服务未启用或连接失败时，相关工具不会提供给 Agent。

关闭服务开关会断开当前连接，但保留服务配置和 Agent 已有的工具选择；再次启用时，系统会重新建立连接并读取工具列表。

点击「编辑」可以修改连接方式及其配置，服务名称保持不变；保存后，已启用的服务会使用新配置重新连接。

「重连全部」会重新连接所有已启用的服务。服务意外断开或连接失败后不会自动重试，需要排查服务状态或修正配置，再手动重连。

删除服务会移除其配置并断开连接，相关工具也会立即变为不可用。

## 为 Agent 选择 MCP 工具

MCP Server 连接成功后，进入控制台并打开 Agent 的配置页面，工具区域会按服务名称列出当前已经发现的 MCP 工具。

展开对应服务后，可以单独选择某项工具，也可以点击「全选」启用当前列出的全部工具。MCP 工具不会因为服务已经连接而自动加入 Agent；新建 Agent 和已有 Agent 都需要明确选择。

「全选」只作用于当前已经发现的工具；如果 MCP Server 之后增加了新工具，需要重新进入 Agent 配置并再次选择。

保存配置后，新的工具选择会从 Agent 下一次执行开始生效。服务暂时断开时，Agent 的选择仍会保留，但相关工具在重新连接前不可使用。

## 配置文件

MCP Server 配置保存在 `data/mcp/servers.json`。该文件已被 Git 忽略，因为环境变量和请求头中可能包含 Token 或 API Key；Git 忽略不等于备份，迁移本地环境时需要单独保留。

通常应通过 MCP 页面管理服务。开发者也可以直接编辑配置文件，运行中的 4torm 会重新读取变更。

```jsonc
[
  {
    "name": "local-tools",
    "enabled": true,
    "transport": "stdio",
    "command": "npx",
    "args": ["-y", "some-mcp-package"],
    "env": {},
    "cwd": "/path/to/server"
  },
  {
    "name": "remote-tools",
    "enabled": true,
    "transport": "streamable-http",
    "url": "https://example.com/mcp",
    "headers": {
      "Authorization": "Bearer YOUR_TOKEN"
    }
  }
]
```

SSE 服务使用与远程 HTTP 服务相同的 `url` 和 `headers` 配置，只需将 `transport` 改为 `sse`。

## 管理接口

MCP 页面通过以下接口管理服务，所有路径均以 `/api/mcp` 开头：

| 方法 | 路径 | 用途 |
|------|------|------|
| `GET` | `/list` | 读取服务配置、连接状态和工具数量 |
| `POST` | `/add` | 添加并连接一个服务 |
| `POST` | `/update` | 更新已有服务的连接配置 |
| `POST` | `/import` | 批量导入已经解析的服务配置 |
| `POST` | `/remove` | 删除指定服务 |
| `POST` | `/toggle` | 启用或停用指定服务 |
| `POST` | `/reconnect` | 重连指定服务；不传名称时重连全部已启用服务 |
| `GET` | `/tools` | 按服务列出当前已经发现的工具 |

添加和更新接口使用“配置文件”一节中的服务结构。批量导入接口接收 `{ "configs": [...] }`；删除、启停和指定重连通过服务名称定位目标。

## 能力边界

4torm 控制的是某项 MCP 工具是否提供给 Agent，并不会替 MCP Server 限制其自身权限。工具能够访问哪些文件、账号或远程数据，取决于对应服务的配置；本地工具的工作区边界不会自动作用于 MCP Server。

当前可以将 MCP 工具返回的文本交给 Agent。图片、音频等非文本结果不会被静默丢弃，但只能返回明确的不支持提示，暂时不能作为内容继续交给模型理解。
