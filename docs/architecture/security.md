# 安全与隔离

4torm 在本地运行，但仍保留文件边界、危险工具约束和敏感数据隔离，减少误操作及配置泄漏。

## 执行权限

Agent 提供「项目级」和「无限制」两种执行权限。该配置会在所有功能区调用框架内置文件工具时自动生效，不需要为季风、对流、气旋、信风或潮汐分别设置。

- **项目级**允许访问 4torm 项目和当前执行所使用的工作区。工作区可以位于 4torm 目录之外；其他未配置的外部路径会被拒绝。
- **无限制**允许内置文件工具访问任意路径。

不同功能区会自动提供自己的当前工作区：对流使用会议室工作区，气旋使用工作室共享目录，信风使用工作流共享目录；季风和潮汐使用 Agent 配置的工作区。

两种权限都禁止普通文件工具直接改写 Agent 注册表、潮汐任务表和工作流控制文件。Agent 记忆、会话内容以及各功能区的工作区文件不属于这类限制。

路径判断会处理相对路径、绝对路径和符号链接。越过当前权限范围时，工具调用会直接失败，并在原有工具卡片中显示错误，不会产生额外确认弹窗。

这套权限只自动约束框架内置文件工具。`run_command` 可以在工作区启动后自行切换目录；MCP 工具和自定义执行器按照自身实现运行，不自动继承文件路径守卫。

## 危险工具约束

写 / 改 / 删文件、执行命令在注册表里标记为危险工具(`dangerous: true`)。它们的约束来自:

- **执行权限** —— 决定内置文件工具可以访问的路径范围(见上)
- **气旋 `plan` 房间** —— 只放行只读工具,自动滤掉危险工具
- `run_command` 内置危险命令黑名单(`rm -rf`、`shutdown`、`mkfs` 等)+ 超时上限

> 早期「逐 Agent always / ask / never」弹窗权限机制已移除,危险工具不再有单独的二次确认弹窗。

## 敏感数据隔离

`data/` 下绝大多数运行时数据都在 `.gitignore` 中、不入仓库,含密钥的尤其敏感:

- `data/providers.json` —— LLM 提供商 API key
- `data/mcp/servers.json` —— MCP 配置(可能含密钥)
- `data/agents/`、`data/cyclone/`、`data/convection/sessions/`、`data/tradewind/`、`data/tide/` —— Agent 注册表与各模式的会话、工作区、运行记录

> 完整清单见[数据目录](./data-layout)页。

## LLM 并发限制

全局最大 **3 路** 并发 LLM 调用(信号量队列),避免对提供商瞬时打满或被限流。超出的调用排队等待。

## MCP 工具名净化

注入 LLM 前,将 `mcp:服务:工具` 等含非法字符的函数名**可逆净化**(兼容 OpenAI 函数命名),LLM 返回工具调用时再还原为原名。既满足函数命名约束,又避免越权 / 串到别的服务器。详见 [MCP 接入](../extend/mcp)。
